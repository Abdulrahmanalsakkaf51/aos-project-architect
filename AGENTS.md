# AOS Project Architect — Working Rules

These rules apply to everyone working in this repository:

- Explain actions simply, especially for a beginner.
- Work one stage at a time.
- Never access files outside this repository.
- Never delete files without approval.
- Never expose or commit API keys or other secrets.
- Ask before making major architectural changes.
- On Windows, use `npm.cmd` or `npx.cmd` if PowerShell blocks `npm.ps1`.
- Run checks after every major coding stage.
- Stop after completing the requested stage.
